# 页面规格文件

## 1. 页面总览
- 项目端类型：admin
- 对应规则集：admin-ui
- 主要角色：导入运营人员；团队演示观看者
- 目标设备/视口：1366x768；兼顾 1440x900
- 页面数量：2 个后台页面
- 主流程：进入表格导入页 -> 选择或查看学校文件 -> 点击“全部开始”或行级“开始导入” -> 查看状态、进度和导入结果 -> 导出问题数据 CSV -> 切换导入历史查看已处理文件。

### 1.1 全局后台壳约定
- 左侧侧边栏：一级菜单，只包含“表格导入”“导入历史”。
- 顶部栏：系统标题、演示环境标识、当前机构、整体状态、重置演示。
- 面包屑：`数据管理 / 当前页面`。
- 内容区：浅灰背景上的白色卡片布局。
- 页面整体：默认 1366x768 下不出现无控制上下滚动；数据多时优先在表格内部滚动。
- 当前演示角色：导入运营人员；不做真实权限系统。

## 2. 页面清单
| 页面 ID | 页面名称 | 所属端 | 服务角色 | 页面用途 | 优先级 | 入口 |
|---|---|---|---|---|---|---|
| A01 | 表格导入 | admin | 导入运营人员 | 选择学校文件、查看统计、执行批量/单文件导入、导出问题数据 | P0 | 左侧菜单“表格导入”；默认落地页 |
| A02 | 导入历史 | admin | 导入运营人员 | 查看状态不是“待导入”的文件列表，复查操作时间与导入结果 | P1 | 左侧菜单“导入历史”；表格导入完成后也可切换查看 |

## 3. 逐页信息架构

### 页面 A01：表格导入
- 所属端：admin
- 页面用途：承载学校数据表格批量导入主流程。
- 入口：左侧菜单“表格导入”；进入系统后的默认页面。
- 返回策略：本页为默认工作页，无需返回按钮；可通过左侧菜单切换到“导入历史”。

页面结构：
1. 全局后台壳
   - 展示内容：左侧深色侧边栏、顶部栏、面包屑和主内容区。
   - 操作：点击左侧“导入历史”切换到 A02；点击“重置演示”恢复初始 mock 状态。
   - 跳转：左侧菜单切换页面，不新开页面。
   - 状态反馈：当前菜单 active；顶部整体状态显示“待导入 / 导入中 / 存在异常 / 全部完成”。

2. 顶部统计卡片区
   - 展示内容：4 个统计卡片，分别为“文件数量”“总记录数”“已入库”“问题数据”。
   - 字段：文件数量、已选中文件数、总记录数、已入库行数、成功率、问题数据总数。
   - 布局：统计区为 1:3 栅格；“文件数量”占 1 份，右侧 3 个指标卡整体占 3 份。
   - 操作：无直接操作。
   - 跳转：无。
   - 状态反馈：导入过程中统计数字随队列状态变化。

3. 文件选择卡片
   - 展示内容：卡片标题“文件选择”；说明“选择检测机构提供的学校表格文件，文件会进入下方导入队列。”；上传区域；文件提示信息。
   - 文件类型：支持 `.xlsx / .xls / .csv`。
   - 操作：点击上传区域选择文件；拖拽文件到上传区域。
   - 交互：选择或拖拽文件后，将文件追加到导入队列；按文件名推导学校名称；不真实解析文件内容。
   - 状态反馈：拖拽悬停态；上传成功提示；导入中禁用上传。

4. 文件导入队列卡片
   - 展示内容：标题“文件导入队列”；说明“队列中的文件可单独开始导入，也可一次性全部开始。”
   - 右上角操作：批量导出问题数据、全部开始。
   - 工具栏左侧：全选、待导入数量、成功数量、异常数量。
   - 工具栏右侧：搜索输入框、状态筛选。
   - 搜索字段：学校名称、文件名。
   - 状态筛选：全部状态、待导入、成功、异常。
   - 表格对象：学校文件导入任务。
   - 表格列：选择框、学校文件、状态、进度、导入结果、说明、操作。
   - 行级操作：开始导入、导出问题数据、删除。
   - 批量操作：全部开始、批量导出问题数据。
   - 状态反馈：待导入、导入中、成功、异常、筛选无结果、无可导出数据提示。
   - 滚动：表格区域占用队列卡片剩余高度；数据多时表格内部滚动。

后台页面补充（仅 admin 页面填写）：
- 侧边栏位置/菜单：左侧固定侧边栏；菜单为“表格导入”“导入历史”；当前 active 为“表格导入”。
- 顶部栏信息：系统标题“云龙区学校数据表格导入平台”；演示环境标签；当前机构“云龙区教育局”；整体状态标签；重置演示按钮。
- 面包屑：`数据管理 / 表格导入`。
- 页面主操作：文件导入队列卡片右上角“全部开始”。
- 页面次操作：批量导出问题数据、选择文件、行级开始导入、行级导出问题数据、删除、重置演示。
- 表格对象：学校文件导入任务。
- 表格列：
  - 选择框：可选中待导入或异常文件；成功文件不可选。
  - 学校文件：文件名；学校名、行数、来源/上传时间。
  - 状态：只向用户展示“待导入 / 成功 / 异常”。
  - 进度：0-100 进度条。
  - 导入结果：成功行数 / 问题数量。
  - 说明：当前处理说明或异常说明。
  - 操作：开始导入、导出问题数据、删除。
- 筛选项：关键词搜索；状态筛选。
- 批量操作：
  - 全部开始：只处理状态为“待导入”的文件；导入中 loading；没有待导入文件时禁用。
  - 批量导出问题数据：导出当前全部问题数据；无问题数据或导入中禁用。
- 行级操作：
  - 开始导入：成功文件禁用；导入中禁用。
  - 导出问题数据：当前文件无问题数据时禁用；导入中禁用。
  - 删除：导入中禁用；删除文件时同步删除关联问题数据。
- 导出/重试/删除等恢复动作：本次不做“重导入已修正数据”；恢复动作通过导出问题数据、删除文件、重置演示表达。
- 页面滚动策略：主内容区在桌面视口内固定高度；统计区固定高度；下方左右工作区占剩余高度。
- 表格内部滚动策略：导入队列表格高度为队列卡片剩余空间；表头保持可见；数据多时只滚动表格内容，不让整页变成长滚动。

不可省略：
- 左右布局必须为 1:3：左侧文件选择、右侧文件导入队列。
- 统计卡片和下方左右布局的间距必须一致。
- 状态对用户只显示“待导入 / 成功 / 异常”。内部排队、校验、导入中只能通过进度和说明表现。
- 不得出现字段自动匹配独立模块、问题数据列表独立模块、导入日志独立模块。
- 问题数据只通过 CSV 导出表达。

### 页面 A02：导入历史
- 所属端：admin
- 页面用途：展示当前队列中状态不是“待导入”的文件，便于复查导入结果。
- 入口：左侧菜单“导入历史”。
- 返回策略：通过左侧菜单切回“表格导入”。

页面结构：
1. 全局后台壳
   - 展示内容：与 A01 共享侧边栏、顶部栏、面包屑和主内容区。
   - 操作：点击左侧“表格导入”切回 A01；点击“重置演示”恢复初始状态。
   - 跳转：左侧菜单切换页面。
   - 状态反馈：当前菜单 active 为“导入历史”；顶部整体状态与队列同步。

2. 导入历史卡片
   - 展示内容：标题“导入历史”；说明“展示当前文件导入队列中状态不是‘待导入’的数据。”
   - 表格对象：已处理学校文件导入任务。
   - 表格列：学校文件、状态、操作时间、进度、导入结果、说明。
   - 操作：本页不提供行级操作；问题数据导出仍从 A01 队列完成。
   - 跳转：无。
   - 状态反馈：无历史数据时显示空态“暂无导入历史。文件完成导入后会出现在这里。”

后台页面补充（仅 admin 页面填写）：
- 侧边栏位置/菜单：左侧固定侧边栏；当前 active 为“导入历史”。
- 顶部栏信息：同 A01。
- 面包屑：`数据管理 / 导入历史`。
- 页面主操作：无。
- 页面次操作：重置演示。
- 表格对象：状态不是“待导入”的学校文件导入任务。
- 表格列：
  - 学校文件：文件名；学校名、行数、来源/上传时间。
  - 状态：成功、异常。
  - 操作时间：文件完成导入或异常确认的时间。
  - 进度：0-100 进度条，历史记录通常为 100。
  - 导入结果：成功行数 / 问题数量。
  - 说明：导入完成说明或异常说明。
- 筛选项：本页不设置筛选，保持简洁。
- 批量操作：无。
- 行级操作：无。
- 导出/重试/删除等恢复动作：无；导出在 A01 的导入队列中完成。
- 页面滚动策略：历史卡片占主内容区；页面整体不做长滚动。
- 表格内部滚动策略：历史表格数据多时内部滚动；空态时保持卡片高度稳定。

不可省略：
- 必须有“操作时间”列。
- 只展示状态不是“待导入”的文件。
- 空态必须说明文件完成导入后才会出现在这里。
- 不得新增导入日志模块或问题数据列表模块。

## 5. 状态与反馈
| 状态名称 | 出现位置 | 触发条件 | 页面表现 | 后续动作 |
|---|---|---|---|---|
| 待导入 | A01 队列表格、统计卡片、顶部状态 | 文件初始状态或重置后 | 状态标签显示“待导入”；进度为 0；说明为待导入 | 可全部开始或行级开始导入 |
| 导入中 | A01 队列表格、顶部状态、按钮 | 点击全部开始或行级开始导入 | 顶部状态显示导入中；按钮 loading/禁用；进度条变化；说明字段显示校验/写入过程 | 等待完成 |
| 成功 | A01 队列表格、A02 历史表格、统计卡片 | 文件无问题导入完成 | 状态标签显示“成功”；进度 100；成功行数等于总行数；问题数为 0 | 进入导入历史；不可再次开始导入 |
| 异常 | A01 队列表格、A02 历史表格、统计卡片 | 文件有问题数据或结构异常 | 状态标签显示“异常”；进度 100；导入结果显示成功行数和问题数；说明给出异常摘要 | 可导出问题数据 CSV |
| 空队列 | A01 队列表格 | 所有文件被删除 | 表格空态提示“暂无文件，请先选择学校表格” | 选择或拖拽文件 |
| 筛选无结果 | A01 队列表格 | 搜索/状态筛选无匹配 | 表格空态提示无匹配文件 | 清空搜索或切换状态 |
| 无导入历史 | A02 历史页 | 没有成功或异常文件 | 空态“暂无导入历史。文件完成导入后会出现在这里。” | 返回 A01 执行导入 |
| 无可导出数据 | A01 批量导出或行级导出 | 当前没有问题数据 | 按钮禁用或点击后提示“当前没有可导出的问题数据” | 等待异常数据产生 |
| 删除成功 | A01 行级删除 | 点击删除且非导入中 | 文件从队列移除，关联问题数据同步删除，提示删除成功 | 继续导入其他文件 |
| 重置成功 | A01/A02 顶部栏 | 点击重置演示 | 恢复初始 mock 队列、清空问题数据、清空搜索筛选、状态回到待导入 | 可重新演示 |

## 7. 实现提醒
- 页面结构实现优先级：先搭后台全局壳与左侧菜单，再实现 A01 主流程，最后实现 A02 历史页。
- 容易遗漏的信息：A02 的“操作时间”列；A01 状态只展示三类；问题数据不做列表，只做导出。
- 需要特别强调的操作：“全部开始”是主按钮；“批量导出问题数据”是次按钮；行级操作为链接按钮或轻量按钮。
- 端适配必须避免的问题：不要使用移动端底部导航；不要做成卡片堆叠的移动端页面。
- 后台页面必须避免的问题：不要增加额外菜单；不要增加字段匹配模块、问题数据列表模块、导入日志模块；不要让整页因表格数据变成长滚动。
- 必须使用图标/媒体/反馈辅助的场景：上传区域需要明确图标或符号；状态标签需要颜色+文字；空态需要简洁提示；导入完成需要通知或消息反馈。
