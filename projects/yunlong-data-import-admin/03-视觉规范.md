# 视觉规范

## 0. 端类型与生效规则
- 项目端类型：admin
- 生效规则集：admin-ui
- 本阶段已读取 reference：`admin-ui/00-后台默认口径.md`、`admin-ui/01-后台信息架构与布局规范.md`、`admin-ui/03-后台表格与筛选规范.md`、`admin-ui/08-现代后台高保真设计系统规范.md`
- 本阶段不改变的 02 结构边界：只包含 A01 表格导入、A02 导入历史；左侧菜单只包含“表格导入”“导入历史”；A01 保持顶部 4 个统计卡片、下方 1:3 文件选择/导入队列；A02 保持历史表格；不新增字段自动匹配、问题数据列表、导入日志模块。

## 1. Design Archetype（设计原型）
### 1.1 内容理解
- 目标用户：导入运营人员、检测机构/教育局侧业务人员、团队演示观看者。
- 核心目的：让用户快速理解学校表格批量导入、状态进度、异常导出、历史查看的后台工作流。
- 期望情绪：可靠、清晰、现代、可控。
- 需避免感受：移动端卡片堆叠、营销页、大屏看板、老旧默认表格模板、装饰压过业务信息。

### 1.2 设计语言
- Aesthetic Direction：现代企业后台，浅灰工作台 + 白色业务卡片 + 深色侧边栏 + 蓝色主操作。
- Visual Signature（3-5 条）：
  - 侧边栏使用深色垂直壳，active 菜单以蓝色轻量高亮。
  - 内容区使用统一 18px 卡片圆角和柔和阴影。
  - 统计区与工作区保持 1:3 栅格节奏，避免宽度断裂。
  - 表格使用浅色表头、stripe 行、pill 状态标签、右侧操作列。
  - 上传区使用虚线边框、浅蓝背景、明确加号/上传符号。
- Emotional Tone：专业、轻快、稳重。
- Design Style：Element 风格的现代后台，但减少默认模板感。
- Application Type：admin system

## 2. Design Principles（设计原则）
- 原则 1：信息架构优先。视觉只强化 02 已定义的后台壳、统计、文件选择、队列表格和历史表格，不新增模块。
- 原则 2：主操作可见。A01 的“全部开始”必须是最突出的主按钮；批量导出和行级操作降低视觉权重。
- 原则 3：状态一眼可辨。待导入、成功、异常必须有文字 + 颜色 + 小圆点，不能只靠颜色。
- 原则 4：表格可读。列宽、行高、表头、操作列保持稳定，不因数据变化挤压关键字段。
- 原则 5：滚动受控。桌面首屏不做无控制整页滚动，队列和历史数据多时只在表格内部滚动。

- Admin Topbar：白色半透明或纯白，底部 `#ebeef5` 细线，文字使用 `#303133`。
- Admin Table Header：`#f7f9fc`，表头文字 `#303133`，不使用深色表头。

### 3.4 色彩防错（Must）
- 主色是否唯一并贯穿按钮/图表主线/重点标签：主色固定 `#409eff`，用于“全部开始”、active 菜单、上传图标、导入中进度主色。
- 是否避免原色/刺眼纯色整卡直铺：状态色只用于标签、图标、小面积提示，不做整卡大面积纯色背景。
- 背景-容器-卡片层级是否可辨：页面 `#f5f7fa`，卡片 `#ffffff`，卡片边框 `rgba(235,238,245,.8)`，阴影 `0 8px 24px rgba(15,23,42,.06)`。

## 4. Typography（字体排版）
- Heading 字体栈：`Inter, PingFang SC, Microsoft YaHei, Arial, sans-serif`
- Body 字体栈：`Inter, PingFang SC, Microsoft YaHei, Arial, sans-serif`
- 字体导入方案：不强制远程字体；优先系统字体，避免网络字体影响原型加载。

### 4.1 层级定义（必须量化）
| 层级 | 字号 | 字重 | 行高 | 场景 |
|---|---:|---:|---:|---|
| 页面标题 | 16px | 760 | 1.25 | 顶部栏系统标题 |
| 区块标题 | 16px | 760 | 1.35 | 文件选择、文件导入队列、导入历史卡片标题 |
| 卡片标题 | 13px | 600 | 1.4 | 统计卡片 label |
| 表格正文（admin 适用） | 14px | 400 | 1.5 | 表格单元格 |
| 正文 | 13px | 400 | 1.55 | 卡片说明、上传说明、工具栏文本 |
| 次要文字 | 12px | 400 | 1.45 | 文件 meta、统计辅助说明、空态说明 |
| 数据数字 | 28px | 760 | 1.15 | 统计卡片数字 |

### 4.2 数字与状态文案规则
- 核心指标数字（首页/统计/详情摘要）：使用 28px、760、tabular nums；不加花哨渐变。
- 状态标签文案长度与截断规则：状态只显示“待导入 / 成功 / 异常”，不展示“排队中/字段校验中/写入中”等内部状态。
- 表格单元格截断/换行规则（admin 适用）：文件名最多两行；说明列使用 tooltip 或单行省略；操作列不换行；元信息使用 12px 次要文字。

## 5. Layout Strategy（布局策略）
### 5.1 结构与导航
- 导航策略（移动端顶部/底部；后台侧边栏/顶部栏）：后台左侧固定侧边栏 + 顶部栏 + 面包屑 + 主内容区。
- 页面架构特征：A01 默认落地页，无额外页面标题区；顶部栏系统标题承担全局标题，面包屑后直接进入统计卡片，遵守 02 不新增结构。
- 主流程首屏信息顺序：面包屑 -> 统计卡片 -> 文件选择/导入队列左右工作区。

### 5.2 响应与密度
- 主验收视口：1366x768；补充 1440x900。
- 断点策略：≥1180px 使用侧边栏 + 1:3 双栏；<1180px 双栏堆叠；<860px 可隐藏侧边栏并保证内容不横向溢出。
- 容器宽度与边距：侧边栏 232px；顶部栏高度 64px；主内容 padding `18px 24px 22px`。
- 模块间距与内部间距：统一外部 gap 18px；卡片 header padding `18px 20px`；表格工具栏 padding `18px 20px 14px`。
- 后台卡片间距统一值（admin 适用）：统计卡片之间、统计区与工作区之间、左右工作区之间均为 18px。
- 表格行高与工具栏高度（admin 适用）：表格单元格垂直 padding 12px；工具栏最小高度 58px；表头高度约 44px。

### 5.3 触控、鼠标键盘与安全区（Must）
- 最小可点击区域：桌面按钮高度不低于 32px；行级文字按钮点击区不低于 28px 高。
- 底部固定区安全间距（mobile 适用）：不适用。
- 鼠标 hover/focus/active 状态（admin 适用）：按钮、菜单、上传区、表格行、行级操作均需 hover；键盘 focus 使用 `outline: 2px solid rgba(64,158,255,.35)`。
- 末项可见性保障方式：表格内部滚动区域底部保留 20px 卡片内边距；空态垂直居中但不撑破卡片。

### 5.4 后台滚动策略（admin 适用）
- 全局壳高度：`height: 100vh; overflow: hidden`。
- 主内容滚动：桌面视口下 `.content-main` 高度为 `calc(100vh - 64px)`，默认 `overflow: hidden`。
- 表格内部滚动：A01 队列表格 flex 填满队列卡片剩余高度；A02 历史表格在卡片内设置固定/自适应高度，数据多时内部滚动。
- 固定顶部/固定操作列策略：表格表头保持可见；A01 操作列可固定右侧；避免固定列阴影过重。

## 6. Visual Language（视觉语言）
### 6.1 形态特征
- 圆角体系：品牌徽章 12px；按钮 10px；提示块 14px；表格容器 16px；业务卡片 18px。
- 阴影体系：卡片 `0 8px 24px rgba(15, 23, 42, 0.06)`；品牌徽章 `0 8px 16px rgba(64,158,255,.25)`；表格内部不再叠加大阴影。
- 描边体系：卡片 `1px solid rgba(235,238,245,.8)`；上传区 `1.5px dashed #cdd6e1`；表格 `1px solid #ebeef5`。

### 6.2 装饰策略
- 可使用的装饰：品牌徽章蓝紫渐变；统计卡右上角浅蓝径向装饰；状态标签小圆点；上传区加号图标。
- 禁止的装饰：营销 hero、背景大渐变、大面积炫彩、卡片套卡片、与业务无关插画、移动端底部导航。

### 6.3 动效与反馈
- 常规过渡时长：150-200ms。
- 按压/选中反馈：按钮 active 轻微变暗；菜单 active 使用蓝色透明背景；上传区 hover/drag 可上移 1px。
- 提交成功/失败反馈：导入完成使用消息/通知；删除成功使用轻提示；无可导出数据使用 warning 提示。
- reduced-motion 处理：进度模拟可保留数值变化，但 hover 位移在 `prefers-reduced-motion` 下关闭。

### 6.4 可及性
- 对比度要求：正文与背景对比至少 4.5:1；次要文字不低于 `#909399`。
- 状态信息不只依赖颜色：所有状态标签必须包含文字；异常说明列必须写原因。
- 键盘/焦点可见性（如适用）：上传区、按钮、筛选输入、状态选择、菜单项均有 focus-visible。

## 7. Core Components（核心组件表现）
- 移动端顶部导航（mobile 适用）：不适用。
- 移动端底部导航/底部操作栏（mobile 适用）：不适用。
- 后台 AppShell（admin 适用）：左侧 232px 深色侧边栏；右侧主容器 `min-width: 0`；整体 100vh。
- 后台 Sidebar（admin 适用）：深色渐变背景；顶部品牌区高度约 64px；菜单 item 高 46px、圆角 12px、间距 6px；active 背景 `rgba(64,158,255,.18)`。
- 后台 Topbar（admin 适用）：高度 64px；白色/半透明白背景；左右布局；右侧包含机构、状态标签、重置演示。
- 面包屑（admin 适用）：位于主内容顶部，字号 13px；与统计卡片间距 14px。
- 卡片：白底、18px 圆角、柔和阴影、轻边框；不要卡片套卡片。
- 列表/时间线：本项目不做独立列表/时间线。
- 表格（admin 适用）：stripe 行；浅色表头；表格外框 16px 圆角；单元格 padding 12px 0；文件名单元格使用主/次两行。
- 筛选/工具栏（admin 适用）：左侧全选和状态数量，右侧搜索和状态筛选；gap 8px；左右对齐，窄屏可换行。
- 表单：不做表单；上传区内隐藏 input，视觉上用上传卡片承载。
- 按钮：主按钮“全部开始”使用 primary；次按钮“批量导出问题数据/重置演示”使用默认或 plain；危险按钮“删除”使用 danger link。
- 标签/状态标识：pill 标签，border-radius 999px，padding `4px 10px`，带 7px 小圆点。
- 弹窗/抽屉：本阶段不定义弹窗/抽屉，05 不得自行新增批次摘要抽屉。
- 空态/异常态/成功态：空态使用虚线边框 + 简洁说明；异常态使用 warning/danger 色标签 + 说明列；成功态使用 success 标签和进度条。
- 图标规则：上传区使用加号或上传图标；菜单可不强制图标；状态标签必须有小圆点。
- 图表规则（主色 token、Tooltip、Legend、容器高度）：不适用，本项目不做图表。

## 8. Page-by-Page Visual Mapping（逐页视觉映射）
> 必须按 `02-页面规格文件.md` 逐页填写；不得用“页面类型模板”替代逐页内容。

### 页面 A01：表格导入
- 所属端：admin
- 对应 02 区域顺序：全局后台壳 -> 顶部统计卡片区 -> 文件选择卡片 -> 文件导入队列卡片。
- 首屏视觉层级：顶部栏和侧边栏建立后台身份；统计卡片展示批次概况；右侧导入队列是视觉重心；“全部开始”是主操作。
- 关键区域视觉重点：
  - 统计卡片：1:3 宽度与下方工作区对齐；数字突出，辅助说明弱化。
  - 文件选择：上传区居中、虚线边框、浅蓝 hover/drag 状态。
  - 导入队列：表格可读优先，工具栏左右布局，状态标签统一。
- 主要操作表现：
  - “全部开始”：primary 按钮，位于队列卡片右上角。
  - “批量导出问题数据”：默认按钮，弱于主按钮。
  - 行级“开始导入/导出问题数据/删除”：link 或轻量按钮，删除使用 danger。
- 状态/反馈表现：
  - 待导入：neutral pill + 小圆点。
  - 成功：success pill + 绿色进度成功态。
  - 异常：warning/danger pill；说明列写原因。
  - 导入中：顶部状态为 primary/info，按钮 loading，进度条动态变化。
- 后台表格/工具栏表现（admin 适用）：
  - 表格列宽稳定；操作列宽约 260px，可固定右侧。
  - 搜索宽度约 220px；状态筛选约 142px。
  - 表格空态在表格内部显示，不撑破布局。
- 滚动与溢出处理（admin 适用）：
  - 主内容高度锁定在 `calc(100vh - 64px)`。
  - 统计区固定高度；下方工作区 flex/grid 填满剩余空间。
  - 队列表格内部滚动，整页不出现无控制长滚动。
- 不可省略视觉点：1:3 栅格、统一 18px gap、状态只显示三类、问题数据只通过导出表达。

### 页面 A02：导入历史
- 所属端：admin
- 对应 02 区域顺序：全局后台壳 -> 导入历史卡片。
- 首屏视觉层级：左侧 active 菜单提示当前位置；历史卡片标题和说明在上；历史表格为核心。
- 关键区域视觉重点：历史表格要稳重、信息密度适中，突出“操作时间”和“导入结果”。
- 主要操作表现：本页无主操作；顶部“重置演示”为全局次操作。
- 状态/反馈表现：
  - 成功历史：success 状态标签，进度 100。
  - 异常历史：warning/danger 状态标签，说明列展示异常摘要。
  - 空历史：虚线边框空态，文案“暂无导入历史。文件完成导入后会出现在这里。”
- 后台表格/工具栏表现（admin 适用）：本页不显示筛选工具栏，避免超出 02 范围；表格列为学校文件、状态、操作时间、进度、导入结果、说明。
- 滚动与溢出处理（admin 适用）：历史表格在卡片内部滚动；空态保持卡片高度稳定；整页不做长滚动。
- 不可省略视觉点：必须有操作时间列；只展示非待导入文件；不新增日志或问题列表模块。

## 9. Quality Gate（质量门）

### 9.1 视觉质量评分（0-100）
- 总分：目标 ≥ 90；低于 85 不得进入 05 实现。
- 评分口径：
  - token 完整性（20）：颜色、字号、间距、圆角、阴影均有可执行值。
  - 层级与信息可读性（20）：后台壳、统计、文件选择、队列表格、历史表格层级清晰。
  - 对比与可访问性（20）：文字对比、状态文字、focus-visible、禁用态清晰。
  - 组件一致性（20）：卡片、按钮、表格、标签、上传区风格统一。
  - 状态反馈与图表/表格可读性（20）：待导入/成功/异常、进度、导出禁用、空态和表格内部滚动明确。

### 9.2 扣分项与原因
- 扣分项：新增 02 未定义模块。
  - 原因：破坏阶段边界。
  - 影响页面/区域：A01/A02 全部。
- 扣分项：页面整体出现长滚动。
  - 原因：违背 02 的滚动策略。
  - 影响页面/区域：A01 文件导入队列、A02 历史表格。
- 扣分项：状态标签只用颜色或状态超过三类展示。
  - 原因：违背用户对状态展示的要求。
  - 影响页面/区域：A01/A02 表格状态列。
- 扣分项：卡片间距不一致或 1:3 对齐失效。
  - 原因：削弱后台页面秩序感。
  - 影响页面/区域：A01 统计区和导入工作区。

### 9.3 必须修复项（Must）
- Must 项：A01 首屏不是标准后台壳。
  - 修复建议：恢复侧边栏、顶部栏、面包屑、内容卡片区。
  - 验收方式：1366x768 截图首屏可见后台壳。
- Must 项：A01 文件选择与队列不是 1:3。
  - 修复建议：桌面栅格设置 `minmax(260px,1fr) minmax(0,3fr)`。
  - 验收方式：统计区与工作区宽度节奏一致。
- Must 项：表格列或操作与 02 不一致。
  - 修复建议：按 02 列清单逐列核对，移除未定义列和未定义操作。
  - 验收方式：A01/A02 表格截图与 02 列清单一致。
- Must 项：问题数据以列表模块出现。
  - 修复建议：删除问题数据列表模块，只保留 CSV 导出入口。
  - 验收方式：页面没有“问题数据列表”独立卡片或 Tab。

### 9.4 交付检查（含低级错误防线）
- 本项目额外禁止事项：不得做移动端样式、营销页、大屏看板；不得新增字段自动匹配模块、问题数据列表模块、导入日志模块；不得新增多级菜单。
- 交付前检查项：
  - 主色是否统一（按钮/图表主线/重点标签一致）：必须统一为 `#409eff`。
  - 页面背景与卡片层级是否可辨：背景 `#f5f7fa`，卡片白底 + 轻边框 + 柔和阴影。
  - 纯白背景或无阴影卡片是否有边界支撑：卡片必须有边框或阴影。
  - 移动端底部固定区是否遮挡末项：不适用；若窄屏适配，不得出现底部遮挡。
  - 触控热区或桌面点击区是否达标：按钮高度不低于 32px，行级操作点击区不低于 28px。
  - 空态/成功/异常态是否为“图标+文案+下一步动作”：空态需有文案；状态标签需小圆点+文字；异常说明列给出原因。
  - 图表主色是否使用项目 token（非默认色）：不适用，本项目不做图表。
  - 后台表格密度、列宽、按钮优先级是否清晰：A01 操作列约 260px，A02 操作时间列约 170px。
  - 后台卡片间距是否统一：统一 18px。
  - 后台页面是否避免无控制整体滚动：1366x768 下整页不长滚动，表格内部滚动。
  - 是否避免明显组件库默认感：通过深色侧栏、18px 卡片、柔和阴影、上传区和 pill 状态标签削弱默认感。
