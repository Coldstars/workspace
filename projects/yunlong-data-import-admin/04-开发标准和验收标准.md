# 开发标准和验收标准

## 1. 开发目标
- 项目端类型：admin
- 生效规则集：admin-ui
- 本次原型用途：演示学校数据表格批量导入平台的后台管理端主流程。
- 主要演示对象：项目团队成员、检测机构业务人员、教育局侧运营人员。
- 实现边界：只实现 A01 表格导入、A02 导入历史、全局后台壳、mock 导入流程、CSV 导出。
- 不在本次范围：真实后端 API、真实 Excel 解析、字段自动匹配独立模块、问题数据列表独立模块、导入日志独立模块、权限管理、多级菜单、批次摘要抽屉。

## 2. 技术实现要求
- 技术栈：React + Vite + CSS。
- Tailwind 方案：不使用 Tailwind，改用 `src/styles.css` 内的 CSS token，确保视觉值与 03 一致。
- 图标库：不引入图标库，使用文本符号和 CSS 小圆点完成上传与状态表达。
- 图表库：不使用图表库。
- 行为组件范围：自定义按钮、卡片、表格、上传区、状态标签、toast/notification。
- 路由方式：不引入路由库，用 React state 在“表格导入 / 导入历史”之间切换。
- 状态管理：React `useState/useMemo` 本地状态。
- 数据来源：本地 mock 数据。
- 主验收视口：1366x768。
- 移动端例外说明：本项目不是移动端；窄屏只做不明显横向溢出保障。
- PC/后台例外说明：桌面视口必须保持后台壳、1:3 栅格、表格内部滚动。

## 3. 代码与数据结构
- 页面目录：`app/src/App.jsx` 承载页面组合；`app/src/main.jsx` 启动。
- 公共组件：在 `App.jsx` 内拆分 `Sidebar`、`Topbar`、`StatCard`、`ImportPage`、`HistoryPage`、`StatusPill`、`Toast`。
- 后台布局壳（admin 适用）：`AppShell` 结构包含侧边栏、顶部栏、面包屑、主内容。
- mock 数据：12 个云龙区学校文件；问题类型包含身份证异常、班级异常、姓名缺失、学校无法识别、文件结构异常。
- 静态资源：不新增图片资源。
- 命名规范：页面 ID 使用 A01/A02；状态内部可用 `ready/queued/matching/importing/success/partial/failed`，展示文案只允许“待导入 / 成功 / 异常”。

## 4. 页面实现范围
- 必须实现页面：A01 表格导入、A02 导入历史。
- 可降级页面：无。
- 主演示链路：默认进入 A01 -> 点击全部开始 -> 查看进度/状态 -> 导出问题数据 -> 切换 A02 查看历史。
- 页面可达性要求：左侧菜单必须能切换 A01/A02，当前菜单 active 清晰。
- 后台侧边栏/顶部栏覆盖要求（admin 适用）：两个页面均共享侧边栏和顶部栏；顶部栏整体状态随导入状态变化。

## 5. 逐页区域实现要点
### 页面 A01：表格导入
- 所属端：admin
- 区域顺序：全局后台壳 -> 面包屑 -> 统计卡片区 -> 文件选择/文件导入队列 1:3 工作区。
- 关键数据：文件数量、选中文件数、总记录数、已入库行数、成功率、问题数据数、学校文件名、学校名、行数、状态、进度、成功行数、问题数、说明。
- 关键交互：点击/拖拽选择文件、全选、搜索、状态筛选、全部开始、单文件开始导入、导出问题数据、删除、重置演示。
- 状态反馈：待导入、导入中、成功、异常、空队列、筛选无结果、无可导出数据、删除成功、重置成功。
- 后台表格/筛选/批量操作（admin 适用）：表格列必须为选择框、学校文件、状态、进度、导入结果、说明、操作；筛选项为关键词和状态；批量操作为全部开始、批量导出问题数据；行级操作为开始导入、导出问题数据、删除。
- 滚动策略（admin 适用）：A01 主内容不做长滚动；统计区固定，下方工作区填满剩余高度；队列表格内部滚动。
- 不可省略点：1:3 布局、18px 统一间距、状态三类展示、问题数据只通过 CSV 导出表达。

### 页面 A02：导入历史
- 所属端：admin
- 区域顺序：全局后台壳 -> 面包屑 -> 导入历史卡片。
- 关键数据：学校文件、状态、操作时间、进度、导入结果、说明。
- 关键交互：左侧菜单进入；重置演示。
- 状态反馈：成功、异常、无历史数据空态。
- 后台表格/筛选/批量操作（admin 适用）：无筛选、无批量操作、无行级操作；只展示非待导入文件。
- 滚动策略（admin 适用）：历史表格在卡片内部滚动；空态保持卡片高度稳定。
- 不可省略点：必须有“操作时间”列；不得新增导入日志或问题数据列表。

## 6. 核心交互
- 页面跳转：左侧菜单切换 A01/A02，不刷新页面。
- Tab / 分段切换：不使用。
- 搜索 / 筛选：A01 支持学校名/文件名搜索；状态筛选包含全部状态、待导入、成功、异常。
- 列表/表格行点击：本次不使用整行点击，避免与行级操作冲突。
- 批量操作（admin 适用）：全部开始只处理 `ready` 文件；批量导出导出全部问题数据。
- 行级操作（admin 适用）：开始导入只处理当前文件；导出问题数据只导出当前文件问题；删除同步清理关联问题数据。
- 导出/下载（admin 适用）：CSV 字段为学校、行号、学生姓名、年级、班级、问题类型、问题字段、当前值、处理建议、状态。
- 表单输入与校验：搜索输入即时过滤；文件 input 只接受 xlsx/xls/csv。
- 提交反馈：导入完成显示通知；删除/上传/重置显示 toast。
- 弹窗 / 抽屉：不实现。
- 图片预览 / 上传预览：不实现；上传只追加 mock 文件。
- 图表切换 / Tooltip / Legend：不实现。
- 空态 / 加载态 / 异常态：队列空态、筛选无结果、历史空态、导入中 loading、异常状态标签必须实现。

## 7. 数据与 mock 规则
- 数据真实感：学校名使用云龙区学校；文件名格式为“学校名学生基础信息表.xlsx”。
- 列表/表格数据数量：初始 12 个学校文件。
- 统计数据：由 `fileJobs` 和 `problemRows` 实时计算。
- 空态与异常态：删除全部文件出现队列空态；未导入时历史为空；异常文件产生问题数据。
- 后台批量操作样本（admin 适用）：全部开始顺序处理所有 `ready` 文件；处理过程模拟排队、校验、写入、完成。
- 导出数据样本（admin 适用）：按问题类型生成 CSV；结构异常生成 1 条 fatal 问题。
- 数据组织：文件任务对象包含 `id/school/fileName/rowCount/issuePlan/fatal/status/progress/successRows/issueCount/uploadedAt/operationTime/message`；问题对象包含 `fileId/school/rowNo/studentName/grade/className/type/field/value/suggestion/status`。

## 8. 实现约束
- 必须遵循的视觉规范：03 中的 `#409eff` 主色、18px gap、18px 卡片圆角、深色侧边栏、浅色表头、pill 状态标签。
- 必须遵循的 reference：`admin-ui/12-React后台实现规范.md`。
- Tailwind 主导样式：不使用 Tailwind。
- 组件库使用边界：不引入组件库；视觉参考 Element 风格但用自定义 CSS 实现。
- 图标与状态：上传区使用加号；状态标签使用小圆点 + 文字。
- 图表实现：无。
- 移动端视口与安全区：不作为主验收；窄屏只需不明显横向溢出。
- 后台布局壳与表格高度（admin 适用）：AppShell `100vh`；Topbar `64px`；Sidebar `232px`；主内容 `calc(100vh - 64px)`；A01/A02 表格内部滚动。
- 性能、交互与无障碍：按钮禁用态明确；focus-visible 可见；状态不只依赖颜色；导入中避免重复触发。
- 本项目额外禁止事项：不得新增字段自动匹配、问题数据列表、导入日志、批次摘要抽屉、多级菜单。

- 会导致演示对象误解的问题：状态展示超过三类、问题数据以列表模块出现、成功/异常统计不一致。
- 会明显拉低高保真感的问题：没有后台壳、卡片间距不一致、整页失控滚动、表格列挤压重叠。
- 后台页面会导致操作不可控的问题（admin 适用）：导入中仍允许删除/重复导入、无问题数据仍允许导出、成功文件可再次开始导入。
- 完成后输出到：`projects/yunlong-data-import-admin/app/`
- 需要同步更新的内容：05 完成后填写截图覆盖矩阵与回归评分。
- 遗留风险：真实 Excel 解析、真实 API、修正后再次导入不在当前范围。
