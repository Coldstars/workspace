# 开发标准和验收标准

## 阶段目标
将 01/02/03 转成 05 可直接执行与复测的标准。

## 输入与输出
输入：`PROJECT.md`、`01`、`02`、`03`、已有 `04`（如存在）。
规则：`workflow/阶段门禁与文档新鲜度规则.md`、`workflow/多页面项目规格组织规则.md`，按端类型读取 `mobile-ui/README.md` 或 `admin-ui/README.md`。
输出：`projects/<project-name>/04-开发标准和验收标准.md`（按 `template.md`）。

## 阶段动作
1. 复核 01/02/03 一致性和端类型。
2. 提取逐页区域顺序、不可省略点、主流程、状态反馈。
3. 定义技术口径（路由/状态/组件边界/mock/目录）。
4. 后台项目必须定义布局壳、侧边栏、顶部栏、表格交互、批量操作、导出、异常态和滚动验收。
5. 输出逐页实现范围与关键交互验收项。
6. 输出核心截图计划与截图覆盖矩阵模板。
7. 输出验收清单与回归评分汇总。
8. 标注 Must 问题与交付边界。

## 阶段质量线
Must：可直接指导 05；验收项可勾选可复测；覆盖可达性、区域还原、交互、状态、构建、截图覆盖矩阵；后台项目覆盖导航壳、表格、批量操作、导出和内部滚动。  
Should：主流程颗粒度更细；风险有优先级。  
Style：清单化短句，避免复述 reference。
