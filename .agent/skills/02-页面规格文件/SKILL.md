# 页面规格文件

## 阶段目标
输出逐页信息架构、页面可达性与状态反馈边界；后台项目还必须锁定导航、表格、批量操作和滚动策略。

## 输入与输出
输入：`PROJECT.md`、`01`、已有 `02`（如存在）。
规则：`workflow/阶段门禁与文档新鲜度规则.md`、`workflow/多页面项目规格组织规则.md`，按端类型读取 `mobile-ui/README.md` 或 `admin-ui/README.md`。
输出：`projects/<project-name>/02-页面规格文件.md`（按 `template.md`）。

## 阶段动作
1. 复核 01 有效性和端类型。
2. 列页面清单（ID/所属端/角色/用途/优先级）。
3. 逐页写区域顺序、操作、跳转、状态。
4. mobile 页面按移动端结构写清导航、触控和安全区。
5. admin 页面按后台结构写清侧边栏、顶部栏、面包屑、页面标题、表格、筛选、批量操作、行级操作、滚动策略。
6. 保证每页可达与可返回。
7. 补齐主流程与高频状态。
8. 输出实现提醒，供 03/04/05 使用。

## 阶段质量线
Must：页面清单与逐页章节一一对应；每页有所属端、区域顺序与不可省略点；所有页面可达；后台表格页必须包含列、筛选、批量操作、行级操作和状态。  
Should：区域命名贴页面语义；主流程闭环可检查。  
Style：逐页可读，不强套统一模板。
