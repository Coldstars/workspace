# 顶部导航规范

## 1. 通用规则
- 顶部导航用于说明当前位置，并承载返回、搜索、筛选、通知、更多等关键入口。
- 移动端页面必须从本文件定义的五类模板中选择一种，不随意发明结构。
- 左侧区域优先放返回、关闭、角色/学校名称；中间区域放标题或对象名称；右侧区域放功能图标。
- 图标使用 `lucide-react`，常规尺寸 `20px`，点击热区遵循 `00-移动端默认口径.md`。

## 2. 首页顶部导航
- 左侧：角色/学校名称。
- 中间：页面标题或问候语。
- 右侧：通知图标。
- 返回按钮：不显示。
- 适用页面：首页、角色工作台、概览页。

## 3. 列表页顶部导航
- 左侧：返回按钮。
- 中间：页面标题。
- 右侧：搜索或筛选图标。
- 适用页面：历史记录、学生列表、班级列表、通知列表、申诉列表。

## 4. 详情页顶部导航
- 左侧：返回按钮。
- 中间：对象名称，例如学生姓名、班级名称、记录编号。
- 右侧：更多或通知图标。
- 适用页面：学生详情、班级详情、扣分详情、记录详情。

## 5. 表单页顶部导航
- 左侧：返回/关闭。
- 中间：表单标题。
- 右侧：保存草稿或空。
- 适用页面：快速检查、申诉申请、扣分确认、信息编辑。

## 6. 统计页顶部导航
- 左侧：返回或角色入口。
- 中间：统计标题。
- 右侧：筛选图标。
- 适用页面：统计分析、趋势页、排行页、风险分析页。

## 7. 实现提醒
- 返回按钮必须有明确返回路径。
- 右侧图标必须在页面规格中说明点击结果。
- 标题过长时使用单行省略，不挤压左右操作区。
- 首页不显示返回按钮，避免演示角色入口和页面返回混乱。
