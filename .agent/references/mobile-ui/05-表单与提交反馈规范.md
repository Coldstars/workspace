# 表单与提交反馈规范

## 1. 表单结构
- 顶部：表单页顶部导航，左侧返回/关闭，中间表单标题。
- 内容：按业务步骤分段，例如对象选择、事项选择、补充信息、附件/取证。
- 底部：固定提交按钮，必要时附带次操作。

## 2. 常见输入
- 搜索：用于选择学生、班级、记录。
- 选择器：用于检查项、班级、时间、状态、类型。
- 单选/多选：用于明确枚举项，不用自由文本替代。
- 文本输入：用于说明、备注、跟进意见。
- 上传/取证：使用占位图、缩略图和删除入口模拟。

## 3. 校验反馈
- 必填未填：字段下方提示，提交按钮触发后同时 Toast。
- 格式错误：在字段附近展示，不跳转页面。
- 禁用态：未满足提交条件时按钮可禁用，但要能看出原因。

## 4. 提交反馈
- 简单成功：使用 Toast 并更新当前页状态。
- 关键流程成功：进入提交成功页，展示编号、对象、状态和通知流转。
- 失败反馈：使用 Toast 或弹窗说明原因，并保留用户已填内容。

## 5. 禁止事项
- 禁止表单项密集堆叠无分组。
- 禁止提交后没有任何反馈。
- 禁止校验提示只写在页面顶部，导致用户找不到错误字段。
- 禁止让移动端用户输入大量可选择的数据。
