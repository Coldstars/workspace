# 列表卡片与信息密度规范

## 1. 列表类型
- 卡片列表：用于检查记录、学生记录、班级排行、申诉事项。
- 分组列表：用于通知、设置项、按日期分组的记录。
- 时间线：用于流转节点、处理进度、通知分发、申诉进度。
- 统计卡：用于首页和统计页的关键数字展示。

## 2. 卡片列表结构
- 主标题：姓名、班级、事项名称或记录编号。
- 副信息：时间、地点、来源、处理人。
- 关键值：分值、数量、排名、风险等级。
- 状态：使用标签或状态点表达，例如待确认、已通知、处理中、已完成。
- 操作：查看、确认、处理、申诉等放在卡片底部或右侧。

## 3. 信息密度
- 移动端单张卡片控制在 3-6 个关键字段。
- 低优先级字段折叠到详情页，不在列表页全部展示。
- 数字、状态、风险优先高亮；说明性文字弱化。
- 列表卡片之间保持稳定间距，不用过重阴影。

## 4. 表格替代规则
- PC 表格列超过 3 列时，移动端必须改为卡片列表。
- 排行表改为排行卡片或分组列表。
- 操作列改为卡片底部按钮或更多菜单。
- 筛选和排序放在顶部筛选区或弹窗中。

## 5. 空态与加载
- 空态必须说明当前无数据原因，并给出下一步动作。
- 加载态使用骨架屏或列表占位，不使用大段文本。
- 异常态只在 mock 原型确有必要时出现，不模拟复杂网络错误。
