# 状态与图标规范

## 1. 图标库
- 默认使用 `lucide-react`。
- 图标风格采用线性图标，保持统一 stroke 风格。
- 常规尺寸：`18px`、`20px`、`24px`。
- 底部导航图标：`22-24px`。
- 图标颜色必须与文本层级或状态色绑定。

## 2. 必须使用图标的场景
- 顶部导航右侧功能入口。
- 底部导航每个 Tab。
- 首页快捷入口。
- 主操作按钮可使用图标增强识别。
- 状态提示：成功、警告、风险、处理中、空态。
- 统计卡可使用轻量图标帮助区分指标。
- 指标卡优先使用 `lucide-react` 中具备明确业务语义的真实图标；没有合适图标时才允许无图标方案。

## 3. 文字图标禁用规则
- 文字图标定义：将汉字、首字、单字、字母、缩写、数字、编号等放入圆形、方形、胶囊底或其他图标容器中，充当图标、头像、入口符号、状态符号或装饰徽标。
- 禁止范围：顶部导航、底部导航、首页快捷入口、指标卡、角色入口、列表卡、状态卡、空态、按钮、头像占位、表单提示、图表图例等所有 UI 样式。
- 允许替代：使用 `lucide-react` 中具备明确业务语义的真实图标；或使用无图标布局、状态色块、细线、圆点、数字层级和排版层级。
- 真实业务文本、按钮文字、字段值、状态文案和数字指标可以正常出现，但不能被设计成装饰图标、头像图标或入口徽标。
- 禁止文字图标不等于禁止图标；不得为了规避文字图标而移除指标卡、入口卡、状态提示等场景中应有的真实业务图标。

## 4. 状态表达
- 成功：使用绿色系状态色，配合 `CheckCircle` 等成功图标。
- 警告：使用黄色/橙色状态色，配合 `AlertTriangle` 等警告图标。
- 风险/告警：使用红色状态色，配合 `CircleAlert`、`ShieldAlert` 等风险图标。
- 处理中：使用蓝色或主色，配合 `Clock`、`Loader` 等状态图标。
- 中性：使用灰色系，配合 `Info` 等信息图标。

## 5. 空态
- 空态必须包含图标、说明和可选行动入口。
- 图标不宜过大，避免压过业务说明。
- 空态文案说明当前无数据原因，不写泛泛的“暂无内容”。

## 6. 成功态
- 关键流程成功页必须展示成功图标、结果编号、关键摘要和下一步操作。
- 成功态要说明系统已经产生了什么变化，例如通知已发送、统计已更新。

## 7. 禁止事项
- 禁止图标风格混用。
- 禁止导航、状态和空态完全无图标。
- 禁止只靠颜色表达风险，必须配合文字或图标。
- 禁止使用无法解释业务含义的装饰图标。
- 禁止任何文字图标。
- 禁止用首字、单字、字母、缩写、数字徽标替代真实图标或无图标方案。
- 禁止将指标卡退化成纯色条或纯文本堆叠来逃避图标设计；应优先选择真实业务图标。
