# 页面布局规范

## 1. 通用移动端页面骨架
- 顶部导航：承载返回、标题、角色/对象名称、搜索、筛选、通知、更多等入口。
- 内容区：采用纵向滚动，安全边距遵循 `00-移动端默认口径.md`，滚动条视觉默认隐藏但滚动能力必须保留。
- 底部区域：根据页面类型使用底部导航或底部固定操作栏，安全区遵循 `00-移动端默认口径.md`。
- 页面宽度：遵循 `00-移动端默认口径.md`，避免在桌面浏览器中拉伸成平板或 PC 布局。

## 2. 首页布局
- 结构：概览指标 + 快捷入口 + 待办/风险 + 最近动态 + 底部导航。
- 首屏目标：让演示对象一眼看懂当前角色、关键数据和下一步动作。
- 常用组件：指标卡、快捷操作宫格、风险提示条、动态列表、底部 Tab。
- 主操作：优先放在首屏中部醒目位置或底部导航的核心入口。

## 3. 列表页布局
- 结构：顶部返回/筛选 + 搜索/筛选区 + 卡片列表 + 空态/加载态。
- 列表数据：优先使用卡片、分组列表或时间线，不直接使用桌面表格。
- 列表项必须包含：主标题、关键字段、状态标签、时间/来源、可点击区域。
- 筛选：高频筛选可以外露，复杂筛选使用弹窗或抽屉。

## 4. 详情页布局
- 结构：顶部返回 + 对象摘要 + 分组信息 + 记录/时间线 + 主操作。
- 对象摘要：放姓名、班级、记录编号、状态、分值等最关键识别信息。
- 分组信息：按业务含义分成卡片，不堆成长段说明。
- 主操作：确认、申诉、跟进、处理等放在底部固定操作栏或详情卡片显著位置。

## 5. 表单页布局
- 结构：顶部返回/关闭 + 分段表单 + 校验反馈 + 底部固定提交。
- 表单分段：按对象选择、事项选择、补充信息、附件/取证组织。
- 输入控件：移动端优先使用选择器、单选、多选、步进器、日期选择、上传占位。
- 提交：主提交按钮固定在底部，禁用态和校验提示必须明确。

## 6. 统计页布局
- 结构：顶部筛选 + 指标卡 + 图表 + 排行/趋势。
- 图表数量：移动端单屏不堆叠过多图表，优先一屏一个主图表。
- 数据强调：关键数字放在指标卡，图表只承担趋势、对比和分布表达。
- 筛选：时间、班级、类型等筛选放顶部或筛选弹窗。

## 7. 禁止事项
- 禁止移动端使用 PC 侧边栏作为主导航。
- 禁止把宽表格压缩到手机屏幕。
- 禁止主操作藏在页面底部内容之后，导致演示时找不到。
- 禁止页面只有文字列表，没有状态、图标和操作入口。
- 禁止移动端原型露出浏览器默认滚动条。
