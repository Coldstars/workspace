# 移动端行为组件映射

## 1. 定位
本文件只定义移动端复杂行为组件的参考方案，不定义默认视觉样式。

默认边界：
- 样式主导：Tailwind CSS。
- 行为组件参考：`antd-mobile`。
- 图标：`lucide-react`。
- 图表：`recharts`，不由 `antd-mobile` 承担。

## 2. 优先自定义的 UI
以下组件优先用 Tailwind 自定义实现，避免组件库默认感：
- 页面容器
- 顶部导航
- 底部导航
- 卡片
- 按钮
- 列表
- 标签/状态标识
- 空态/成功态/告警态
- 图表外层卡片和图例容器

## 3. 可参考 antd-mobile 的行为组件
- `Picker`：复杂选择器。
- `DatePicker`：日期/时间选择。
- `Popup`：底部筛选、底部表单。
- `Dialog`：确认、危险操作二次确认。
- `Toast`：轻提示和次要操作反馈。
- `Tabs`：页面内分类切换。
- `Switch`：开关。
- `ImageUploader`：上传/取证模拟。
- `ActionSheet`：少量动作选择。

## 4. 表单行为映射
- 表单结构可参考 `Form`，视觉由 Tailwind 统一。
- 输入可参考 `Input`、`TextArea`，但必须与项目卡片、边框、字号一致。
- 搜索可参考 `SearchBar`，也可自定义 Tailwind 搜索框。
- 单选/多选可参考 `Radio`、`Checkbox`、`Selector`。
- 上传/取证可以用 `ImageUploader` 或自定义缩略图占位。

## 5. 反馈行为映射
- 简单反馈：`Toast`。
- 确认反馈：`Dialog`。
- 底部选择/筛选：`Popup`。
- 动作选择：`ActionSheet`。
- 加载态：`Skeleton` 或自定义骨架。
- 空态：自定义空态组件，配合 `lucide-react` 图标。

## 6. 实现底线
- 不为移动端原型引入 PC 组件库替代移动端行为组件。
- 不使用桌面 `Table` 承载移动端列表。
- 不让底部固定操作区遮挡内容。
- 不让 `antd-mobile` 默认皮肤替代项目视觉规范。
- 不把卡片、按钮、导航、列表和状态组件全部交给组件库默认样式。
