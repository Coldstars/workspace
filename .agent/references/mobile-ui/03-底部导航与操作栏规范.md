# 底部导航与操作栏规范

## 1. 底部导航
- 用于一级模块/一级空间切换，例如首页、班级、记录、消息、我的。
- 底部导航不承载详情页、表单页、结果页、统计详情、申诉详情等次级页面。
- 建议数量：3-5 项。
- 每项包含图标和短标签，图标使用 `lucide-react`，尺寸 `22-24px`。
- 当前项必须有清晰选中态，颜色与主色绑定。
- 底部导航固定在页面底部，并按 `00-移动端默认口径.md` 处理安全区。
- 底部导航项不得与首页快捷入口、Hero 主按钮、顶部通知入口表达同一跳转目标。

## 2. 底部固定操作栏
- 用于提交、确认、继续处理、查看详情等当前页面主操作。
- 主按钮触控高度遵循 `00-移动端默认口径.md`。
- 操作栏必须避免遮挡页面末尾内容，内容区底部需要预留操作栏高度。
- 表单页和详情页的关键确认操作优先使用底部固定操作栏。

## 3. 主次操作位置
- 主操作：放底部固定按钮、首屏快捷入口或卡片显著位置。
- 首页快捷入口只放高频动作、跨模块捷径或临时任务，不复制底部导航 Tab。
- 次操作：放顶部右侧、卡片内链接、弹窗内或更多菜单。
- 危险操作：必须二次确认，不直接放在底部主按钮。
- 成功后：必须有 Toast、结果页或状态更新反馈。

## 4. 安全区规则
- iOS 底部安全区遵循 `00-移动端默认口径.md`。
- 底部导航与底部固定操作栏不可同时抢占同一位置；如果共存，操作栏应位于导航上方或改为页面内按钮。
- 页面主体底部 padding 必须大于底部固定区域高度。

## 5. 禁止事项
- 禁止底部按钮遮挡最后一条列表或表单项。
- 禁止底部导航超过 5 项。
- 禁止只有文字没有图标的底部导航。
- 禁止把表单页、详情页、结果页、统计详情、申诉处理详情放进底部导航。
- 禁止首页快捷入口与底部导航重复同一批页面入口。
- 禁止把低频操作做成底部主按钮。

## 6. 规格验收
- 页面规格生成后必须检查底部导航、首页快捷入口、Hero 主按钮、顶部通知入口是否重复。
- 如果某功能已经是底部导航一级空间，首页不得再用快捷宫格重复入口。
- 如果某功能是一次性任务或流程动作，应放在 Hero、快捷入口或底部操作栏，不应放在底部导航。
