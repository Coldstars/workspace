# 移动端默认口径

## 1. 适用范围
本文件是 `.agent/references/mobile-ui/` 的唯一默认口径来源。其他移动端参考规范只描述各自领域规则，不重复完整默认值。

默认服务对象：
- 移动端 Web 高保真交互原型。
- 桌面浏览器中的手机宽度演示容器。
- 以 Codex 实现为目标的 React 前端原型。

PC 后台、大屏、原生 App 如需偏离本口径，必须在 03 视觉规范或 04 开发标准中写明原因。

## 2. 默认技术栈
- 前端：`React + Vite`
- 样式：`Tailwind CSS`
- 图标：`lucide-react`
- 图表：`recharts`
- 移动端行为组件参考：`antd-mobile`

组件边界：
- Tailwind 主导视觉样式。
- `antd-mobile` 只作为复杂移动端行为组件参考，例如 Picker、Popup、Dialog、Toast、Tabs、Switch、ImageUploader。
- Recharts 负责统计页主图表。
- `antd-mobile` 不负责视觉、不负责图表、不负责卡片/导航默认样式。

## 3. 默认视口与触控
- 主验收视口：`375x812`、`390x844`
- 页面最大宽度：`430px`
- 页面左右安全边距：`16px`
- 主按钮触控高度：不低于 `44px`
- 图标按钮点击热区：不低于 `44x44`
- 底部区域：必须处理 `safe-area-inset-bottom`

## 4. 默认实现底线
- 不接真实后端，使用本地 mock 数据。
- 页面之间必须可点击跳转。
- 表单、筛选、Tab、弹窗、Toast、图表切换等关键交互必须可演示。
- 主流程优先于边角功能。
- 页面不得横向滚动。
- 移动端滚动容器必须隐藏可见滚动条，但不得禁用内容滚动；局部横向滚动列表、Tabs 等可滚动组件也必须隐藏滚动条视觉。
- 移动端列表不得直接照搬桌面表格。
- 导航、状态、空态、统计卡必须有统一图标辅助。
- 组件库默认视觉不得替代项目设计系统。
- **严禁渲染模拟手机状态栏**（信号格、WiFi 图标、电量、顶部系统时间条）。本项目为浏览器演示的移动端 Web 原型，页面顶部必须直接从顶部导航栏开始，不得用任何 div/组件模拟 iOS/Android 系统状态栏。
  - 问题案例：Agent 在页面顶部插入含「9:41 · 信号 · 电量」的灰色/深色条，导致视觉廉价且与真实业务页面结构不符。
  - 预期收益：消除伪状态栏，页面直接从顶导开始，更像企业级移动 Web。
  - 影响范围：全部 21 个页面的顶部实现。
  - 验证方法：截图检查，首屏最顶部是否直接是顶部导航组件（背景色 + 标题 + 返回/右侧图标），不含任何系统仿制元素。
  - 回滚条件：如用户明确要求演示 App 效果图时，可在 03 视觉规范中申报例外。

## 5. 规则优先级
1. 项目已确认文档中的明确约束。
2. 本文件默认口径。
3. 同目录下的专项参考规范。
4. 具体实现中的合理工程约束。

当专项规范与本文件冲突时，以本文件为准，除非项目文档明确说明例外。
