# UI 质量预设与实践

## 1. 定位
本文件是 03/04 阶段的 UI 质量入口，只保留优先级和最小要求。配色模板详见 `附录-配色模板库.md`，需要选色时再读取。

## 2. 质量优先级
1. 无障碍：对比度、焦点、语义、键盘可达、图标按钮名称。
2. 触控与交互：44px 热区、8px 间距、press/focus/active/loading 反馈。
3. 性能感知：图片/图表/异步内容预留空间，避免布局跳动和明显卡顿。
4. 风格一致性：产品类型匹配、语义色固定、图标风格统一。
5. 移动端结构：无横向滚动，固定导航和底部操作不遮挡内容。
6. 动效：150-300ms，用于解释状态变化，并尊重 `prefers-reduced-motion`。
7. 表单反馈：可见 label、错误就近、提交反馈和恢复路径。
8. 图表：legend、tooltip、空态、加载态和可访问颜色。

## 3. 03 视觉规范最小要求
- 选择一个配色模板，或说明自定义原因。
- 输出语义色 token，不在页面级散落 hex。
- 写明主色/状态色使用比例和对比度注意点。
- 将无障碍、交互反馈、性能感知纳入交付检查。

## 4. 04 开发与验收最小要求
- 性能、交互、无障碍必须变成可执行约束和验收项。
- 图片、图表、异步内容必须有稳定尺寸、宽高比或 skeleton。
- 关键控件必须有 press/active/focus-visible。
- 图标按钮必须有 `aria-label` 或可见文字。
- 表单必须有可见 label 和就近错误提示。
- 状态表达不能只依赖颜色，必须配合图标、文案或形状。

## 5. 禁止事项
- 随机 hex、随机阴影、随机渐变。
- 移除焦点样式。
- hover-only 作为移动端关键入口。
- 图标按钮无可访问名称。
- 图片、图表或异步内容加载后明显撑开页面。
