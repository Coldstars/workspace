# 移动端 UI 参考规范库

## 用途
提供移动端全局规范；项目业务上下文只来自 `projects/<project-name>/`。

## 引用策略
- 项目阶段文档只写：生效规则、例外、验收方式。
- 同一规则优先单点引用，不在 01~05 重复改写。
- 规则冲突时，先更新对应阶段文档，再实现。
- 术语口径以 workflow 的“统一术语字典”为准。

## 读取地图
### 02 页面规格
- 必读：`00`、`01`
- 按需：`02`、`03`、`04`、`05`

### 03 视觉规范
- 必读：`00`、`08`、`11`
- 按需：`附录`、`02`、`03`、`04`、`06`、`09`、`10`

### 04 开发与验收
- 必读：`00`、`09`、`11`
- 按需：`03`、`05`、`06`、`07`、`10`

### 05 原型实现
- 必读：`00`、`12`
- 按需：`09`、`10`、`11`

## 读取上限建议
- 默认：2 个必读 + 最多 3 个按需。
- 超上限时，先输出“本次生效规则清单”再补充扩展。

## 文件职责
- `00` 默认口径；`01-07` 页面/交互专项；`08` 设计系统；`09` Tailwind；`10` 图表；`11` 质量；`12` 实现规范。
