# 后台组件尺寸与控件规范

## 适用范围
用于后台管理端的按钮、输入框、选择器、表格、进度条、标签和卡片标题区。目标是让 03/04/05 阶段有稳定默认尺寸，避免实现阶段凭感觉反复调整。

## 默认控件基线
- 后台默认字号：`14px`。
- 默认控件高度：按钮、输入框、选择器、搜索框使用 `32px`。
- 默认圆角：按钮、输入框、选择器使用 `6px`；卡片按 08 规则使用 `12-16px`。
- 默认边框：`1px` 低对比边框，hover/focus 必须有清晰反馈。
- 默认行内图标：`16px`，与文字间距 `6-8px`。

## 按钮规则
- 每个操作组最多 1 个主按钮；其余使用 default、plain、text 或 link。
- 页面主操作放在页面标题区或当前业务模块工具栏，不在多处重复。
- 表格行级操作优先使用 text/link 样式，避免把每行做成一排实心按钮。
- 危险操作使用 danger，并在可能造成数据丢失时提供确认反馈。
- 按钮文字保持短句，不在按钮内写解释性长文案。

## 输入、选择器与搜索框
- 默认高度 `32px`，字号 `14px`，圆角 `6px`。
- 搜索框 placeholder 使用 14px，不得大于正文。
- 筛选控件宽度应稳定，避免输入后工具栏跳动。
- hover 使用边框色增强，focus 使用边框色加轻量 focus ring。
- 重置、展开筛选、刷新等次级动作不得抢主按钮视觉权重。

## 表格组件
- 表格正文：`14px`，常规字重。
- 表头：`14px / 600`，背景轻量，不做厚重色块。
- 默认行高：`44-48px`。
- 表格内进度条：`6-8px`，不得做成厚重进度块。
- 状态标签：`12px`，pill 或轻量描边样式，状态文案短而稳定。
- 操作列：按钮间距一致，危险操作与普通操作有清晰区分。
- 数据多时优先使用表格内部滚动，避免整个页面无控制滚动。

## 卡片 Header
- 卡片标题区默认不加横线。
- 标题与说明之间用字号、颜色和间距区分，不依赖分割线。
- 只有高密度详情页、配置页、审计页确实需要分区时，才允许使用低对比细线。
- 卡片之间的外间距必须统一；同一页面不得一会儿 12px、一会儿 24px 随意变化。

## 时间格式
- 后台列表、历史、日志默认使用 `YYYY-MM-DD HH:mm`。
- 秒级时间只有在业务明确要求排查精确顺序时才使用。
- 不同页面展示同一类时间字段时，格式必须一致。

## 验收检查
- 按钮、输入框、选择器是否落在 `14px / 32px` 默认口径。
- 表格行高、表头、进度条和状态标签是否统一。
- 卡片标题区是否默认无横线。
- 操作组是否只有一个主按钮。
- 时间字段是否使用统一格式。
