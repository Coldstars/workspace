# 现代后台高保真设计系统规范

## 设计方向
后台视觉应服务于效率、可信度和可读性。优先现代、清晰、稳定，避免炫技和过度装饰。

## Token 建议
- 主色：用于主按钮、active 菜单、重点状态，不到处泛滥。
- 状态色：success、warning、error、info 必须区分明确。
- 背景：页面浅灰，内容白色或近白。
- 边框：低对比细线，辅助分区。
- 圆角：卡片通常 12-16px；表格、按钮按项目风格统一。
- 阴影：柔和、低透明度，用于层级，不制造漂浮感。

## 成熟后台组件默认尺寸
- 后台默认正文、按钮、输入框、选择器字号使用 `14px`。
- 默认按钮、输入框、选择器、搜索框高度使用 `32px`。
- 控件默认圆角使用 `6px`，hover/focus 必须有清晰但克制的边框反馈。
- 页面主按钮遵循“1 个主按钮 + n 个次按钮”，避免多个主按钮并列争夺注意力。
- 表格正文 `14px`，表头 `14px / 600`，默认行高 `44-48px`。
- 表格进度条使用 `6-8px` 细条，不做厚重进度块。
- 状态标签使用 `12px`，样式稳定、文案短，不用大号标签占据表格行高。

## 布局质感
- 卡片间距统一。
- 页面标题区与内容区留白清晰。
- 卡片标题区默认不加横线，除非高密度配置或审计页面需要明确分区。
- 表格表头轻量，行高稳定。
- 工具栏左右布局：左侧批量操作，右侧搜索、筛选、刷新等。
- 统计卡片与下方内容的栅格比例应稳定，避免宽度节奏断裂。

## 控件质感
- 参考 Ant Design / Element Plus 的企业后台控件节奏：轻边框、清晰 hover/focus、稳定尺寸。
- 搜索框、选择器、按钮在同一工具栏内应高度一致。
- 表格行级按钮优先使用 text/link 样式；危险操作使用 danger。
- 控件文案使用操作语言，避免在按钮内写说明书。

## 禁止
- 大面积单色渐变压过业务内容。
- 把后台页做成营销 hero。
- 卡片套卡片造成层级混乱。
- 表格按钮过多且无优先级。
- 页面滚动、卡片间距、表格高度临时拼凑。
- 为装饰添加无业务意义的图表或图片。

## 视觉验收
- 首屏能识别这是后台系统。
- 主要动作 3 秒内可定位。
- 表格列与状态可读。
- 卡片间距、按钮优先级、状态标签统一。
- 按钮、输入框、选择器是否保持 `14px / 32px` 默认口径。
- 卡片标题区默认无横线；表格进度条为细条。
- 页面没有明显横向溢出、重叠、截断和无控制滚动。
