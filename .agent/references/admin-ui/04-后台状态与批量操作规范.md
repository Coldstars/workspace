# 后台状态与批量操作规范

## 状态最低集合
后台页面至少考虑：
- 默认态。
- 加载态。
- 空态。
- 处理中。
- 成功。
- 失败。
- 部分成功。
- 筛选无结果。
- 不可操作/权限不足。
- 可导出/不可导出。
- 可重试/不可重试。

## 批量操作要求
批量操作必须定义：
- 入口位置。
- 可操作对象范围。
- 禁用条件。
- 确认方式。
- 进度反馈。
- 结果摘要。
- 失败项处理。
- 是否支持导出或重试。

## 结果反馈
- 成功：明确已处理数量和下一步。
- 部分成功：明确成功/失败数量，提供失败项查看或导出。
- 失败：明确失败原因和恢复动作。
- 异常数据：必须能定位、导出或重试。

## 验收
- 每个主操作都有开始、进行中、结束反馈。
- 失败路径不是死路，有重试、导出、返回或重置。
- 状态信息不只依赖颜色。
