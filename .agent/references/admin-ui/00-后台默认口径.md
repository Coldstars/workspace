# 后台默认口径

## 目标
后台管理端原型用于给团队、客户或业务方演示管理流程，而不是一次性生成真实生产系统。

## 默认假设
- mock first，真实 API later。
- 桌面优先，默认验收视口为 1366x768 与 1440x900；如项目需要，再补窄屏视口。
- 企业后台风格：稳定、清晰、克制、可读优先。
- 页面必须可点击、可演示、可解释。
- 本地 mock 数据要接近未来接口形状，包含正常、异常、空态和批量处理样本。

## 不做
- 不接真实后端，除非用户明确要求。
- 不把后台做成营销页、大屏看板或移动端页面。
- 不在实现阶段新增 02 未定义的菜单、页面、字段、按钮。
- 不用装饰性视觉块替代信息架构。

## 验证
- PROJECT 明确 `端类型: admin` 或 `mixed`。
- 门禁表显示已读取 `admin-ui`。
- 02/03/04 能回答后台页面的导航、模块、表格、操作、状态和验收。
