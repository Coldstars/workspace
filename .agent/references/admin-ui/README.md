# 后台管理端 UI 参考规范库

## 用途
提供后台管理端全局规范；项目业务上下文只来自 `projects/<project-name>/`。

## 引用策略
- 项目阶段文档只写：生效规则、例外、验收方式。
- 同一规则优先单点引用，不在 01~05 重复改写。
- 规则冲突时，先更新对应阶段文档，再实现。
- 术语口径以 workflow 的“统一术语字典”为准。

## 读取地图
### 02 页面规格
- 必读：`00`、`01`、`02`、`03`
- 按需：`04`、`05`、`06`、`07`

### 03 视觉规范
- 必读：`00`、`01`、`07`、`08`
- 按需：`02`、`03`、`04`、`05`、`09`、`10`

### 04 开发与验收
- 必读：`00`、`03`、`04`、`07`、`12`
- 按需：`05`、`06`、`08`、`09`、`10`

### 05 原型实现
- 必读：`00`、`01`、`07`、`12`
- 按需：`03`、`04`、`05`、`08`、`09`、`10`

## 读取上限建议
- 默认：3 个必读 + 最多 3 个按需。
- 后台表格页必须读取 `03` 与 `04`。
- 有图表模块必须读取 `09`；有图标库或图片素材必须读取 `10`。
- 超上限时，先输出“本次生效规则清单”再补充扩展。

## 文件职责
- `00` 默认口径；`01` 信息架构与布局；`02` 页面模块；`03` 表格与筛选；`04` 状态与批量操作；`05` 表单弹窗抽屉；`06` 导航与权限；`07` 组件尺寸与控件；`08` 设计系统；`09` 图表与可视化；`10` 图标与图片资源；`12` React 实现规范。
