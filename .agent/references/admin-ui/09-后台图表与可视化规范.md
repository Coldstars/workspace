# 后台图表与可视化规范

## 适用范围
用于后台管理端的数据看板、趋势分析、分布统计、批量处理结果和业务监控页面。图表必须服务具体业务问题，不作为装饰性元素自动添加。

## 默认技术口径
- 复杂仪表盘、趋势、堆叠、联动图表优先使用 Apache ECharts。
- 只有 `02` 明确存在图表模块，且 `04` 允许新增依赖时，才在实现阶段引入图表库。
- 简单数字概览优先使用统计卡片，不为了“显得高级”强行加图表。

## 数据组织
- 图表数据优先采用 dataset 风格：数据与图表配置分离。
- mock 数据结构应贴近未来接口结构，避免把业务字段硬编码进渲染配置。
- 同一页面多图表共享筛选条件时，筛选状态要集中管理。

## 图表基础元素
- 图表必须定义标题或上下文说明，让用户知道回答的业务问题。
- 图例、Tooltip 是理解图表的关键组件；除非图表极简单，否则必须提供。
- 图表必须定义空态、加载态、无数据态。
- 图表颜色必须复用 03 token，不使用 ECharts 默认彩虹色。
- 数字单位、日期粒度、排序口径必须可见或可推断。

## 默认尺寸
- 小卡片图表：`180-220px`。
- 标准分析图表：`280-360px`。
- 高密度看板高度由 `04` 单独定义，并纳入截图验收。
- 图表容器需要稳定高度，加载、空态、数据态之间不得明显跳动。

## 交互与可读性
- Tooltip 内容要使用业务语言，不直接暴露内部字段名。
- Legend 可点击筛选时，需要保持布局稳定。
- 多序列图表必须有清晰颜色区分，不只依赖颜色时应增加文本或图例说明。
- 图表与表格联动时，联动结果需要有可见反馈。
- 截图验收必须覆盖至少一个有数据态；如图表可为空，也要覆盖空态。

## 禁止
- 没有业务问题就添加图表。
- 使用默认彩虹色、强渐变或过度发光效果。
- 用图表替代表格中的关键可核对数据。
- 只做静态装饰图，无法解释数据来源和指标含义。
