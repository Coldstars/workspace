# 后台页面模块规范

## 常见模块
后台页面按业务需要组合以下模块：
- 页面标题区：标题、说明、主操作、次操作。
- 指标概览：数量、状态、风险、进度。
- 筛选区：搜索、状态、时间、组织、标签等。
- 工具栏：批量操作、导出、刷新、新增。
- 数据表格：列、状态、进度、行级操作。
- 详情区：基础信息、关联记录、处理记录。
- 表单区：录入、编辑、校验、提交。
- 异常处理：问题列表、修复入口、重试、导出。
- 历史/日志：操作时间、操作者、动作、结果。

## 模块书写规则
02 页面规格中，每个模块必须写清：
- 模块目标。
- 展示字段。
- 主要操作。
- 状态反馈。
- 与其他模块的联动。

## 不可省略
- 主操作的位置与禁用条件。
- 次操作与危险操作的优先级。
- 空态和异常态的下一步动作。
- 演示所需的 mock 数据类型。
