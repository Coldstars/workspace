# React 后台实现规范

## 默认技术口径
- React + Vite 优先。
- 本地 mock 数据优先。
- 不新增依赖，除非 04 明确允许。
- 默认可引入 `lucide-react` 作为后台图标库；如需行业图标且 04 允许，可用 Tabler Icons 补充。
- 有图表需求且 04 明确允许依赖时，可引入 `echarts` 或 React 封装。
- 不为了装饰随意引入图库图片；图片来源、用途和授权口径必须在 04 写清。
- 代码结构按页面复杂度拆分：`views/`、`components/`、`mock/`、`types/`、`utils/`。

## 实现顺序
1. 建立 AppShell：侧边栏、顶部栏、主内容容器。
2. 建立路由或页面切换，让页面清单全部可达。
3. 实现 token 与全局样式。
4. 实现 mock 数据与主流程状态。
5. 实现页面模块：标题、概览、筛选、表格、表单、详情、异常、历史。
6. 补充空态、加载、异常、重试、导出等状态。
7. 执行构建、交互和截图验收。

## 依赖边界
- 图标库：默认 `lucide-react`；Tabler Icons 只作为补充，不建议同页混用两套风格。
- 图表库：默认 Apache ECharts；只有存在图表模块时才安装。
- 图片素材：默认不使用；确需使用时优先 Unsplash、Pexels，并遵守 10 的敏感场景限制。
- 所有新增依赖必须在 04 的允许依赖清单中出现，05 不得临时追加未声明依赖。

## 后台页面硬门槛
- 页面必须有明确后台壳，不得像独立移动端卡片页。
- 表格列、按钮、状态必须来自 02，不得自行增删。
- 批量操作必须有反馈。
- 导出必须有可演示行为或明确 mock 提示。
- 数据多时表格或内容区内部滚动，不让整个页面失控滚动。
- 控件尺寸遵循 07：按钮、输入框、选择器默认 `14px / 32px`。
- 卡片标题区默认不加横线；表格进度条使用 `6-8px` 细条。
- 图表必须有业务问题、Tooltip、Legend、空态和 token 色板，不使用默认彩虹色。
- 图片不得暗示真实人物、学校、机构背书或敏感身份。
- 截图覆盖至少包含主列表页、表单/详情页、异常或空态。

## 截图建议
- 桌面主视口：1366x768。
- 宽屏视口：1440x900。
- 窄屏回归：如 04 要求，补 1024x768 或移动窄屏。

## 质量评分维度
- D1 结构与层级：后台壳、页面标题、模块顺序、表格结构。
- D2 质感与风格：token、间距、圆角、阴影、现代感。
- D3 业务语义：字段、状态、操作、mock 数据是否可信。
- D4 排版容错：无重叠、无截断、无横向溢出、滚动策略正确。
- D5 专项资源：控件尺寸、图标一致性、图表可读性、图片授权与业务适配。

任一维度低于 4/5，或总分低于 20/25，不得宣称视觉验收通过。
