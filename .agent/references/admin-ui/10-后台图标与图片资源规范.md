# 后台图标与图片资源规范

## 默认图标库
- 默认使用 Lucide React：线性风格、SVG、可 tree-shake，适合作为 React 后台默认图标库。
- Lucide 缺少行业图标时，可使用 Tabler Icons 补充。
- 同一页面原则上不混用两套图标风格；必须混用时，统一线宽、尺寸和视觉重量。

## 图标尺寸
- 表格行级操作图标：`16px`。
- 按钮内图标：`16px`。
- 侧边菜单图标：`18px`。
- 顶部工具栏图标：`18-20px`。
- 空态、异常态图标：`40-56px`。
- 数据概览或大图标入口：`24-32px`，不得喧宾夺主。

## 图标风格
- 优先线性图标，统一 `24x24` 视图盒。
- stroke 推荐 `1.75-2px`，同一区域保持一致。
- 图标必须辅助识别，不替代关键文字。
- 图标按钮必须有可访问名称或可见文字。
- 状态表达不得只靠图标，必须同时有文本或标签。

## 图片素材来源
- 免费图片来源优先使用 Unsplash、Pexels。
- 图片来源、用途和授权口径必须在 `04` 写清；实现阶段不得临时随意引入图库图片。
- 后台页面默认不使用装饰性大图。
- 只有用户、机构、场景、证件、设备、空态表达确实需要时才引入图片。

## 教育与敏感场景限制
- 禁止使用可能暗示真实儿童、学生、机构背书或敏感身份的照片。
- 教育类项目优先使用抽象办公、文件、数据、设备、校园环境局部等低风险图片。
- 不使用含清晰人脸、校牌、证件号、身份证件细节的图片作为演示素材。
- 不得让图片暗示某个人、学校、机构认可或背书当前产品。

## 验收检查
- 图标库是否在 04 允许依赖清单中声明。
- 图标尺寸、线宽和风格是否统一。
- 图片是否有明确业务用途和来源。
- 图片是否存在人物背书、敏感身份或教育场景误导风险。
- 页面是否把图片当成默认装饰，而不是服务业务表达。
