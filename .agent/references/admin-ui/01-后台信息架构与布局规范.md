# 后台信息架构与布局规范

## 全局布局
后台页面默认包含：
- `AppShell`：全局壳。
- `Sidebar`：一级导航或分组导航。
- `Topbar`：系统名、组织/角色、全局操作、用户入口。
- `Breadcrumb`：当前位置。
- `PageHeader`：页面标题、说明、主操作和次操作。
- `MainContent`：卡片、表格、详情、表单等业务内容。

## 页面层级
推荐顺序：
1. 面包屑。
2. 页面标题区。
3. 概览或关键状态。
4. 筛选/工具栏/主操作。
5. 主内容表格或详情。
6. 异常处理、历史记录、日志或补充信息。

## 滚动策略
- 页面整体应避免无控制长滚动。
- 数据密集表格应优先使用表格内部滚动或内容区滚动。
- 侧边栏和顶部栏固定时，主内容高度必须显式约束。
- 卡片之间间距必须统一，不能因局部布局变更破坏节奏。

## 导航要求
- 侧边栏菜单必须与页面清单对应。
- 当前页面必须有清晰 active 状态。
- 页面可达路径必须写进 02。
- mixed 项目不得用移动端底部导航替代后台侧边栏或顶部栏。
